4 · The 8 workflows in detail

Mapping the brief's workflow blueprint to our implementation.

WF 1 — Source-discovery scrapers
Cron · weekly Mon 06:00
Brief asked for: Nightly Rightmove / Bayut / Indeed / LinkedIn Jobs scrape.
We built: after5/sources/ — DuckDuckGo (most reliable), Indeed UK (best-effort, often blocked), Bayt & GulfTalent (UAE — currently disabled), Clutch agency search, hiring-signal search.

The blueprint document framed scraping as the lead-acquisition layer. We layered in CSV import as the primary intake — drop a curated list at /upload and it imports into the pipeline. Scraping is the backup when no curated list is available.

WF 2 — 6-Signal qualifier
DB trigger · status=new
Brief asked for: 6 signals (tech, SEO, Maps reviews, ads, hiring, sentiment), binary 0/1 each, 3/6 minimum to enter sequence.
We built: Each signal is a separate module under after5/scrapers/. Each returns a 0-10 raw score; the qualifier turns that into binary 0/1 at threshold ≥3. qualify.py sums binaries and buckets into hot/warm/cold.

The 6 signals — what each one detects

tech.pyHTTP headers + JS-tag fingerprint. Shopify/WordPress/HubSpot/Wix → low-friction install. Intercom/Drift → already pay for chat → easy upsell.
seo.pyDuckDuckGo site: probe — counts indexed pages to gauge SEO footprint.
reviews.pyTrustpilot JSON-LD parse — review count + average rating.
ads.pyMeta Ad Library API (free, official) + Google Ads Transparency Center.
hiring.pyProbes /careers, /jobs; keyword-sniffs for SDR / BDR / customer success / sales / marketing roles.
sentiment.pyPulls Trustpilot review text → Ollama/Groq classifies positive/neutral/negative. Hits HIGH score if multiple reviews mention "slow response" / "no answer" — exactly our pain.
WF 3 — Contact enrichment
DB trigger · status=qualified
Brief asked for: Clay/Apollo lookup for Founder + Head of Sales + Head of Marketing; verify via listclean.
We built: contacts.find_for_company() — Hunter.io free tier (25/mo) filtered by role keyword. Missing roles get role-pattern fallback (founder@, sales@, marketing@) with MX validation.

Honest limit: Hunter's 25/mo cap means we'll run out fast at the brief's 2,000 companies/month volume. For now, the pattern-fallback covers everything else. Bounce parsing in WF8 catches the bad ones after-the-fact.

WF 4 — AI personalisation
DB trigger · new contact row
Brief asked for: Claude Haiku via AWS Bedrock writes a 1-sentence opener referencing the strongest signal.
We built: ai.py tries Groq first (free, Llama 3.1 70B — markedly better than Haiku for this task) → falls back to local Ollama → graceful empty string if neither available.

Example output (real, generated against demo data):

"Saw you're hiring SDRs at deliveroo.co.uk — usually a sign the team's stretched."

WF 5 — Send sequencer
Cron · hourly · business hours
Brief asked for: SmartLead Pro 90k sends/month, 14-day sequence, daily caps per inbox, UK 8-10am send time.
We built: send.py — Gmail SMTP with day 1/4/12/30/60/90 sequencer. Daily cap, per-domain suppression, MX probe, bounce capture, List-Unsubscribe header. Dry-run mode for safe testing.
WF 6 — Reply triage
Webhook · IMAP poll
Brief asked for: LLM classifier → interested / asking_info / objection / not_now / unsubscribe / ooo, then route.
We built: triage.py reads Gmail IMAP, classifies with Groq/Ollama into 5 labels (interested, not_interested, ooo, unsubscribe, other). Interested replies set needs_louis=1 and ping Slack.

Gap: Brief splits "asking_info" and "objection" into distinct labels. We currently merge into "other". Easy to expand when needed.

WF 7 — Loom reminder
Cron · every 6 hours
Brief asked for: If an interested reply is older than 48h with no Loom sent, Slack-DM Louis.
We built: loom.py — exactly that. Idempotent via slack_pinged_at column so Louis only gets pinged once per reply. Plus a "Mark Loom sent" button on the dashboard's Replies page.
WF 8 — Daily dashboard
Cron · 8am daily
Brief asked for: 8am Slack + email digest of yesterday's pipeline metrics.
We built: Persistent web dashboard with live KPIs, recent leads, recent activity feed, "needs Louis" alert tile. Daily Slack ping after send completes. Email digest not yet built.
+ Manual touch tracker (extra)
UI · click to log
Not in brief but added because: LinkedIn invites / cold calls / Looms can't be automated safely (ToS / no free alternatives).
We built: touches table + per-contact log form on company detail page. Tracks: kind (linkedin_invite / linkedin_voice / cold_call / loom / other), day number, status, notes. So nothing falls through the cracks even though the action is manual.

6 · Dashboard tour

Sidebar (left)

  • Dashboard — KPI funnel, recent leads, top sources, recent activity, pipeline-by-campaign tiles.
  • Leads — every company. Filter by country / status / priority / campaign. Click any to see signal scores, contacts, sends, and manual touches.
  • Import Leads — drag a CSV (curated list from AI Ark or any source) into the pipeline.
  • Contacts — every individual. Tabs for ready-to-send / verified / sent / unsubscribed.
  • Preview Emails — see exactly what would go out next for every ready contact, rendered with their company name and signal.
  • Sent Emails — chronological archive of every email ever sent. Each is expandable to show the rendered body.
  • Templates — read-only viewer for the 6 email templates (day 1, 4, 12, 30, 60, 90). Edit via git.
  • Replies — classified inbox with "Needs Louis" filter; Mark-Responded / Mark-Loom-Sent / Snooze actions.
  • Automation — all 10 pipeline stages as manual-trigger buttons + cron schedule reference.
  • Setup — env var status cockpit + cron URL generator + DNS record helper.

The primary actions on the dashboard

  • "Import Leads" (green button) — open /upload, drop a CSV. Most common entry point for the AI Ark intake flow.
  • "Run Pipeline" — chains discover → enrich → qualify in one click. Useful when you want to top up the funnel and don't have a fresh CSV.
  • "Automation" — opens the full job-trigger panel.

7 · Going live — what Louis provides

The pipeline is wired and waiting. To switch from demo to real outreach, set these in Render's Environment tab:

Must-have (free)

  1. Gmail sending account — your real sending email + a 16-char App Password (generated at myaccount.google.com/apppasswords with 2-Step Verification on).
    Set: SMTP_USER, SMTP_PASS, SENDER_NAME, REPLY_TO.
    Also: enable IMAP in Gmail settings.
  2. Slack incoming webhook — free at api.slack.com/messaging/webhooks. Lets the pipeline DM you when an interested reply lands.
    Set: SLACK_WEBHOOK_URL.
  3. Groq API key — free at console.groq.com. Enables AI personalisation in the cloud (Llama 3.1 70B).
    Set: GROQ_API_KEY.
  4. Hunter API key (optional) — free 25 lookups/month at hunter.io. Gets real decision-maker emails vs role-pattern guesses.
    Set: HUNTER_API_KEY.

Cheap-but-needed (paid)

  1. Sending domain at Porkbun — ~£12/year. Don't send cold from a personal Gmail; sender reputation tanks fast.
  2. SPF / DKIM / DMARC — generate the exact records at /setup/dns?domain=<yours> and paste into Porkbun's DNS panel.
  3. Google Workspace on the new domain — £4.60/mo Starter plan. Gives you the real DKIM key + a clean sending inbox.
  4. Render Starter — $7/month. Removes the 15-min sleep + persistent disk so the DB doesn't reset on every deploy.

Cron schedule (free)

To run the pipeline unattended without paying for Render's Background Worker, configure cron-job.org with these POST endpoints:

POST /cron/pipeline-intake     # Weekly Mon 06:00 — top up the funnel
POST /cron/triage              # Daily 09:00 — classify replies
POST /cron/bounces             # Daily 09:00 — suppress bad addresses
POST /cron/send-live           # Daily 09:00 — send today's batch
POST /cron/loom-check          # Every 6 hours — nudge stale interested

All require header X-Cron-Token: <your CRON_TOKEN> — value is on the /setup page.

8 · Honest gaps

GapWhyFix
LinkedIn auto-actions ToS prohibits automation; accounts get banned. Manually do them; log in touches table. Louis was always going to do these by hand per the brief.
Auto-dialler No safe free alternative to CloudTalk. Same — manual + tracked.
900/day from one inbox Gmail caps free at ~500/day, Workspace at ~2,000 with reputation throttling. Brief's 15 domains × 3 inboxes math is correct. Start with 1 × 1, scale.
Domain warmup Manual ramp 5/day → 50 → 200 over 2-4 weeks is unavoidable for deliverability. Cannot be scripted; just don't skip it.
SQLite ephemeral on Render free DB resets each deploy on the free tier. Upgrade to Render Starter ($7/mo) for persistent disk, OR migrate to Postgres on Neon (free).
Listclean-grade verification Their data is proprietary; SMTP RCPT-TO probing is unreliable (Gmail catch-alls everything). Live with bounce parsing + MX check. Acceptable for our volume.
Documented Legitimate Interest Assessment (UK GDPR) Brief doesn't include one; cold email under UK GDPR requires a written LIA. Louis or his team needs to write a 1-page LIA before going live to UK SMEs.
